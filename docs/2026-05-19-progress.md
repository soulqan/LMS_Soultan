# Progress Update - 2026-05-19

## Summary

Today I tightened the public account flow and the admin user management flow. The LMS now has clearer role handling, an editable profile page for students, and stronger privacy boundaries inside Filament.

## What Changed

- Added a `role` column to users with `student` as the default for public registration.
- Introduced two roles:
  - `student`
  - `admin`
- Updated the Filament user resource so it only manages admin accounts.
- Removed the role filter from the Filament users table.
- Added an editable student profile page and changed the header action from `My Courses` to `Profile`.
- Kept `Sign In` as the only guest header action and removed the separate sign-up button from the header.
- Added a logout button to the shared header so signed-in users can end their session cleanly.

## Privacy Update

- Admins can no longer browse student accounts in the Filament user resource.
- The user resource query is scoped to `admin` accounts only.
- Student registration still exists on the public site and continues to create `student` users by default.
- The public profile page lets students update their own name, email, and password, but not their role.
